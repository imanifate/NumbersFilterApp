NumbersFilter – مرحله اول

در مرحله اول پروژه، یک نسخه ساده و پایه‌ای از برنامه ساخته شده که هدف آن دریافت مجموعه‌ای از اعداد از کاربر، ذخیره‌سازی آن‌ها و در نهایت نمایش اعداد بزرگ‌تر از مقدار مشخص (در این مرحله: ۵) است.

این مرحله تمرکزش روی ایجاد ساختار اولیه برنامه و فراهم کردن مقدمه‌ای برای قابلیت‌های پیشرفته در مراحل بعدی است.


Purpose :

- جداسازی منطق برنامه از ورودی/خروجی (Console)
- جداسازی مسئولیت‌ها (Separation of Concerns)
- آماده‌سازی کد برای امکان نوشتن Unit Test
- استفاده از Fake در Unit Test برای شبیه‌سازی Console
- جلوگیری از وابستگی مستقیم به `Console`
- استفاده از متدهای `static` برای ساده‌سازی نسخه اولیه

## 🧱 معماری مرحله اول
در این مرحله از **سه کلاس اصلی** استفاده شده:

### 1️⃣ `Messages`
این کلاس فقط پیام‌های ثابت برنامه را نگه‌داری می‌کند تا کدنویسی تمیز، قابل تست و قابل تغییر باشد.

### 2️⃣ `ConsoleUserIO`  
کلاسی برای مدیریت **ورودی و خروجی**، که رفتار اصلی Console را در خود نگه می‌دارد.  
دلیل این جداسازی:
- جلوگیری از استفاده مستقیم از `Console` در منطق برنامه
- قابلیت جایگزینی آسان ورودی/خروجی در Unit Test
- افزایش Testability

### 3️⃣ `NumberService`
این کلاس شامل منطق اصلی کار:
- خواندن ورودی کاربر
- تبدیل ورودی به عدد
- ذخیره‌سازی در List 
- فیلتر کردن اعداد
- نمایش خروجی

**تمام متدهای این مرحله به‌صورت `static` نوشته شده‌اند** تا نسخه پایه سریع و ساده باشد.

Features :

دریافت ورودی از کاربر تا زمانی که "done" وارد شود

مدیریت کامل ورودی/خروجی بدون وابستگی مستقیم به Console
- تبدیل رشته به عدد و بررسی صحت ورودی استفاده از (TryPars)
- فیلتر اعداد بزرگ‌تر از ۵
- نمایش پیام‌ها و نتایج به‌صورت منظم و قابل تست
- استفاده از xunitTest و 
- قابلیت تست کامل بدون استفاده از Console
-استفاده از **FakeConsoleUserIO** برای شبیه‌سازی I/O

## 📉 محدودیت‌های مرحله اول (Limitations)
- فیلتر ثابت است (بزرگ‌تر از ۵
- الگوریتم فیلتر ساده و محدود
- استفاده از متدهای `static` (که در مراحل بعدی حذف می‌شود)
- در این مرجله فقط از list[int] پستیبانی میکند


How to Run : 

پروژه را اجرا کنید.
اعداد را وارد کنید.
برای پایان، "done" را تایپ کنید.
خروجی شامل اعداد بزرگ‌تر از ۵ نمایش داده می‌شود.
